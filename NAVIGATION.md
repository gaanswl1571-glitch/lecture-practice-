## vault 구조 철학

**종류 먼저, 월별 비교 가능하게.**
- 자료를 찾을 때 "어떤 종류인지"로 먼저 접근
- 월별로 쌓이는 자료를 나중에 비교하기 쉽게 날짜를 파일명에 포함
- 내가 만든 것 vs 받은 것을 명확히 구분

---

## 폴더 구조

```
vault/
├── NAVIGATION.md              ← 지금 이 파일 (vault 지도)
│
├── Mine/                      ← 내가 직접 만든 자료
│   ├── 연구개발비/            ← 월별 예산 실적 보고서
│   │     예) 2026-03_연구개발비_보고서.md
│   │     예) 2026-04_연구개발비_보고서.md
│   ├── PPR/                   ← PPR 보고 파일 정리
│   │     예) 2026-Q1_PPR_요약.md
│   └── 기획/                  ← 연구 기획서, 제안서
│
├── Library/                   ← 받은 자료 (타 부서 등)
│   └── 타부서-참고자료/        ← 부서명_날짜_제목 형식으로 저장
│         예) 재무팀_2026-03_예산지침.md
│
└── Archive/                   ← 완료·종결된 자료 보관
```

---

## 파일 이름 규칙

| 종류 | 규칙 | 예시 |
|------|------|------|
| 연구개발비 보고서 | `YYYY-MM_연구개발비_보고서` | `2026-04_연구개발비_보고서.md` |
| PPR | `YYYY-QN_PPR_요약` | `2026-Q1_PPR_요약.md` |
| 타 부서 자료 | `부서명_YYYY-MM_제목` | `재무팀_2026-03_예산지침.md` |

> 날짜를 파일명 앞에 붙이면 → 정렬했을 때 자동으로 시간순 정렬 + 월별 비교 쉬워짐

---

## 월별 비교 팁

월별 연구개발비 보고서를 비교하고 싶을 때:

```
@Mine/연구개발비/ 폴더 안의 3월, 4월 보고서를 비교해서
달성률 차이와 원인을 표로 정리해줘
```
